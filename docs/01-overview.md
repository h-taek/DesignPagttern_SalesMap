# 01. 프로젝트 개요

## 주제

**서울시 소상공인 매출 및 예측 시각화 지도 시스템**

지도 기반으로 서울시 자치구(25개)의 업종별 매출 정보를 직관적으로 조회하고, 과거 데이터를 기반으로 선형회귀 기반 **분기 매출 예측**을 제공한다.

> 그레인은 `자치구 × 분기 × 업종 대분류 3개(외식/서비스/소매)`. 데이터 소스인 OA-15572가 분기 단위이므로 PDF 발표 자료의 "월별·구/동" 표현은 "분기·구"로 좁혀 진행한다. 자세한 배경은 [04-data-model](04-data-model.md) 참고.

## 개발 동기

1. **지역별 매출 흐름 파악의 중요성**
   - 소상공인의 생존율 제고를 위해 객관적인 상권 데이터 분석이 필요하다.
2. **기존 공공데이터의 직관성 부족**
   - 표 형태의 데이터로는 지역 간 비교나 트렌드 파악이 어렵다.
3. **의사결정 보조 도구 필요**
   - 과거 데이터에 더해 예측 데이터를 함께 제공하여 미래 전략 수립을 지원한다.

## 주요 기능

| # | 기능 | 설명 |
|---|------|------|
| 1 | 자치구 지도 표시 | 서울시 25개 자치구 폴리곤 렌더링 |
| 2 | 지역·업종 선택 | 마우스 오버 / 클릭으로 구역 식별 + 업종(외식/서비스/소매) 토글 |
| 3 | 매출 정보 연동 | 백엔드 API를 통한 분기 매출 + 예측 데이터 수신 |
| 4 | 시각화 대시보드 | 팝업창과 분기 추이 차트로 결과 표시 |

## 데이터 및 예측 방식

- **소스**: 서울 열린데이터 광장 OA-15572 (상권분석서비스 추정매출-상권). 자세한 호출/매핑은 [08-external-api](08-external-api.md).
- **저장**: 외부 데이터를 자치구 × 분기 × 업종 대분류로 사전 집계해 SQL DB에 저장. 예측 결과도 같은 DB에 보관하고 사용자 요청 시 단순 조회로 반환.
- **모델**: 선형회귀 모델로 과거 분기 매출의 추세를 학습한다.
- **배치 예측**: 사용자 요청 시 실시간 학습/예측을 수행하지 않고, 분기 단위 배치 작업으로 미리 예측 결과를 계산해 저장한다. 서버 부하를 줄이고 응답 속도를 개선하기 위한 선택이다.

## 사용자 시나리오

1. 사용자가 웹 URL로 접속한다.
2. 서울시 25개 자치구 지도가 표시된다.
3. 사용자가 업종(외식/서비스/소매)을 선택하고 특정 자치구를 클릭한다.
4. 선택한 `(regionId, industry)`가 백엔드로 전송된다.
5. 백엔드는 DB에서 최신 분기 매출과 다음 분기 예측을 조회해 반환한다.
6. 프론트는 팝업과 분기 추이 차트로 결과를 시각화한다.

## 관리자 시나리오 (배치)

1. 스케줄러(n8n)가 분기 시작 시점에 배치를 트리거한다.
2. **Step 1**: n8n → BE `POST /api/ingest/sales`. BE가 OA-15572에서 신규 분기 데이터를 받아 자치구 × 업종 대분류로 집계 후 `sales_record`에 upsert.
3. **Step 2**: n8n → AI `POST /predict/batch`. AI 서버가 셀(`region × industry`) 단위로 선형회귀를 학습하고 다음 분기 매출을 예측해 `prediction_record`에 저장.
